1 Simple Explanation

Linear algebra is the mathematics of vectors and matrices — organized grids of numbers that you can add, multiply, and transform. If you've ever used a spreadsheet with rows and columns of numbers, you've already thought in linear algebra. In AI, everything is represented as numbers arranged in these grids: images, words, audio, and even the parameters of a neural network itself.

A vector is a list of numbers, like [0.2, -0.5, 0.8]. A matrix is a 2D grid of numbers. A tensor is the generalization to any number of dimensions — a 3D tensor might be a batch of images (batch × height × width). When a neural network processes your input, it is doing nothing more than a long chain of matrix multiplications and additions.

💡 Key insight: Every forward pass through a neural network is matrix multiplication. Every embedding lookup is a vector. Every attention score is a dot product. Linear algebra is not background knowledge for AI — it is AI at the mathematical level.

2 Why It Matters for AI Engineers

When you see a shape mismatch error in PyTorch (RuntimeError: mat1 and mat2 shapes cannot be multiplied), linear algebra tells you exactly what went wrong and how to fix it. When you want to measure how similar two embeddings are, the dot product is your tool. When you need to understand why your model's gradients are exploding, eigenvalues explain it. You don't need to be a mathematician — but you need the core operations cold.

3 Real-World Example

When OpenAI's text-embedding-3-large model converts your sentence into a 3072-dimensional vector, that vector lives in a high-dimensional vector space. Finding similar sentences means computing cosine similarity — a normalized dot product — between vectors. The entire semantic search industry (Pinecone, Weaviate, Chroma) is built on one linear algebra operation. Separately, Google's PageRank algorithm — which ranked the entire web — is an eigenvector computation. And PCA, used to compress embeddings before storing them in vector databases, is singular value decomposition (SVD) under the hood.

4 Step-by-Step: Core Operations

Vectors — The Building Block

python
import numpy as np

# A word embedding is just a vector
king   = np.array([0.9, 0.1, 0.8, 0.2])
queen  = np.array([0.8, 0.9, 0.7, 0.3])
man    = np.array([0.7, 0.1, 0.6, 0.1])
woman  = np.array([0.6, 0.9, 0.5, 0.2])

# Famous vector arithmetic: king - man + woman ≈ queen
result = king - man + woman
print(result)   # [0.8, 0.9, 0.7, 0.3] ≈ queen ✓

# Vector magnitude (length)
magnitude = np.linalg.norm(king)
print(f"Magnitude: {magnitude:.3f}")

# Unit vector (normalize to length 1)
king_unit = king / magnitude
print(f"Unit vector magnitude: {np.linalg.norm(king_unit):.3f}")  # 1.0

Dot Product & Cosine Similarity

python
import numpy as np

def cosine_similarity(a: np.ndarray, b: np.ndarray) -> float:
    """
    The standard way to measure embedding similarity.
    Returns 1.0 = identical direction, 0.0 = orthogonal, -1.0 = opposite.
    """
    dot_product = np.dot(a, b)          # scalar: how aligned are they?
    norm_a = np.linalg.norm(a)
    norm_b = np.linalg.norm(b)
    return dot_product / (norm_a * norm_b)

doc1 = np.array([1.0, 0.5, 0.2, 0.8])
doc2 = np.array([0.9, 0.4, 0.3, 0.7])   # similar
doc3 = np.array([-0.8, 0.1, -0.5, 0.0])  # different

print(cosine_similarity(doc1, doc2))  # ~0.99 — very similar
print(cosine_similarity(doc1, doc3))  # ~-0.6 — quite different

Matrix Multiplication — The Core of Neural Networks

python
import numpy as np

# A single linear layer in a neural network is just: output = input @ W + b
batch_size, input_dim, output_dim = 4, 8, 3

X = np.random.randn(batch_size, input_dim)   # 4 samples, 8 features each
W = np.random.randn(input_dim, output_dim)   # weight matrix
b = np.random.randn(output_dim)              # bias vector

output = X @ W + b   # matrix multiply then add bias
print(output.shape)   # (4, 3) — 4 samples, 3 outputs each

# Shape rule: (m×n) @ (n×p) → (m×p)
# Inner dimensions MUST match. Outer dimensions become the result.
print(f"X: {X.shape}, W: {W.shape} → output: {output.shape}")
# X: (4,8), W: (8,3) → output: (4,3)  ✓ inner dims both = 8

# Transpose — flip rows and cols (used constantly)
W_T = W.T
print(W_T.shape)   # (3, 8)

Eigenvalues & SVD — Used in PCA and Embeddings

python
import numpy as np

# PCA via SVD — compress 1536-dim embeddings to 2D for visualization
np.random.seed(42)
embeddings = np.random.randn(100, 1536)   # 100 documents, OpenAI embedding size

# Center the data
centered = embeddings - embeddings.mean(axis=0)

# SVD decomposes matrix into U, S, Vt
U, S, Vt = np.linalg.svd(centered, full_matrices=False)

# Project to 2D using top 2 singular vectors
embeddings_2d = centered @ Vt[:2].T
print(embeddings_2d.shape)   # (100, 2) — ready to plot!

# Variance explained by each component
variance_explained = (S**2) / np.sum(S**2)
print(f"Top 2 components explain: {variance_explained[:2].sum():.1%} of variance")

  [sample 1]           [col 1 ]          [out 1]
  [sample 2]     ×     [col 2 ]    =     [out 2]
  [sample 3]           [col 3 ]          [out 3]
  [sample 4]                             [out 4]

  Rule: (m×n) @ (n×p) → (m×p)
         inner dims must match ↑↑

COSINE SIMILARITY — What it looks like geometrically

       ↑
    B  │  A         cos(θ) ≈ 1.0  →  nearly identical direction
       │ /          cos(θ) = 0.0  →  completely unrelated
       │/ θ         cos(θ) ≈ -1   →  opposite meaning
       └──────→

  your query vector vs every stored document vector.

6 Practical Use Cases

Semantic search: Every RAG pipeline converts queries and documents to vectors then ranks by cosine similarity. The entire retrieval step is a batch of dot products between a query vector and a matrix of stored embeddings.

Dimensionality reduction: OpenAI's embedding API lets you request 256-dimensional embeddings instead of 1536 by applying PCA (SVD) internally. This cuts storage costs by 6× with minimal accuracy loss.

Attention mechanism: The Q·Kᵀ in self-attention is a matrix multiplication producing a (seq_len × seq_len) attention score matrix. Understanding this shape is essential for reasoning about context window costs.

7 Common Mistakes

❌ Mistake 1: Confusing shape order. PyTorch is row-major: shape (batch, seq, dim). Transposing the wrong axes causes silent bugs where math still runs but produces nonsense outputs.
❌ Mistake 2: Using Euclidean distance instead of cosine similarity for embedding comparison. Embeddings often have different magnitudes — cosine similarity normalizes for this. Raw L2 distance will give wrong rankings.
❌ Mistake 3: Not centering data before PCA. SVD on uncentered data doesn't give you principal components — it gives you directions from the origin. Always subtract the mean first.
❌ Mistake 4: Forgetting that @ and np.dot behave differently on batched inputs (3D+ tensors). Use torch.bmm or torch.einsum for batched matrix ops in PyTorch.
