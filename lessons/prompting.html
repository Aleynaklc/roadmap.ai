1 Simple Explanation

Prompt engineering is the art and science of communicating with AI models effectively. Just like how you'd phrase a request differently to a junior intern vs. a senior expert, the way you phrase a request to an LLM dramatically affects the quality of the output.

💡 Key insight: A well-engineered prompt can make a smaller model outperform a larger one on specific tasks. Prompt engineering is often the highest-leverage skill an AI engineer can have — it's free compute.

2 Core Techniques

Zero-Shot, One-Shot, Few-Shot

python
from openai import OpenAI
client = OpenAI()

def chat(prompt: str) -> str:
    return client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": prompt}]
    ).choices[0].message.content

# ─── ZERO-SHOT: No examples ───
zero_shot = """Classify the sentiment of this review as 
POSITIVE, NEGATIVE, or NEUTRAL.

Review: "The product arrived late but worked great!"
Sentiment:"""

# ─── FEW-SHOT: Give examples first ───
few_shot = """Classify sentiment as POSITIVE, NEGATIVE, or NEUTRAL.

Examples:
Review: "Amazing quality, fast shipping!" → POSITIVE
Review: "Broke after one day, terrible." → NEGATIVE  
Review: "It's okay, nothing special." → NEUTRAL
Review: "Love it! But delivery was a day late." → POSITIVE

Now classify:
Review: "The product arrived late but worked great!"
Sentiment:"""

# Few-shot almost always outperforms zero-shot on classification
print(chat(few_shot))

Chain-of-Thought (CoT) Prompting

python
# Standard prompt — often wrong on math/logic
standard = """A store sells apples for $1.50 each and oranges for $2.00.
If Sarah buys 3 apples and 2 oranges, and pays with a $10 bill,
how much change does she get?"""

# Chain-of-Thought — "think step by step"
cot = """A store sells apples for $1.50 each and oranges for $2.00.
If Sarah buys 3 apples and 2 oranges, and pays with a $10 bill,
how much change does she get?

Let's think step by step:"""

# Even simpler CoT trigger:
cot_simple = "... Think step by step before giving your final answer."

# Self-consistency: run CoT multiple times, take majority vote
def self_consistent_answer(question: str, n: int = 5) -> str:
    answers = [chat(question + "
Think step by step.") for _ in range(n)]
    # In practice, extract just the final answer and vote
    return max(set(answers), key=answers.count)

System Prompts & Persona Engineering

python
# Anatomy of a production system prompt
SYSTEM_PROMPT = """You are CodeReviewer, an expert senior software engineer 
specializing in Python, security, and performance optimization.

## Your Persona
- Direct and technical — no fluff
- Always cite specific line numbers when giving feedback
- Security-first mindset: flag any potential vulnerabilities

## Your Task
Review submitted code and provide structured feedback.

## Output Format
Always respond in this exact JSON structure:
{
  "summary": "One sentence overall assessment",
  "issues": [
    {"severity": "critical|high|medium|low", "line": N, "description": "..."}
  ],
  "suggestions": ["..."],
  "score": 1-10
}

## Rules
- Never write code for the user unless asked
- If the code has a security vulnerability, always rate it critical
- Be specific, not vague"""

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": SYSTEM_PROMPT},
        {"role": "user", "content": "Review this: eval(user_input)"}
    ],
    response_format={"type": "json_object"}
)

3 Advanced Patterns

ReAct Pattern (Reason + Act) — Have the model reason before acting. Used in agents.
TREE-OF-THOUGHT — Explore multiple reasoning paths, like a tree search.
Structured Output — Force JSON output and parse it reliably with Pydantic.
Prompt Chaining — Break complex tasks into sequential prompts where output of one feeds the next.
