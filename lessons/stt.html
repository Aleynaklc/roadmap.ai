1 Simple Explanation

Speech to Text (STT) — also called Automatic Speech Recognition (ASR) — converts spoken audio into written text. When you dictate a message, talk to a voice assistant, or see auto-generated captions on a video, STT is running underneath. For AI engineers, it is the critical first step in any voice-enabled application: it converts the human voice into a format that language models can reason about.

The current gold standard is OpenAI's Whisper, a Transformer-based model trained on 680,000 hours of multilingual audio. It accepts raw audio, processes it as a mel spectrogram — a visual representation of sound frequencies over time — and outputs a transcript including punctuation, timestamps, and automatic language detection.

The architecture works in two stages. First, an audio encoder (CNN + Transformer) converts the mel spectrogram into a rich sequence of audio features. Then, a text decoder auto-regressively generates the transcript token by token — exactly like GPT generating text, but conditioned on audio features instead of previous text tokens.

💡 Key insight: Whisper is a multitask model. The same model can transcribe audio in 99 languages, translate non-English audio directly to English, and identify which language is being spoken — all without retraining. This is achieved by prepending special task tokens to the decoder at inference time.

2 Why It Matters for AI Engineers

Voice interfaces are the fastest-growing input modality for AI. Building voice assistants, meeting transcription services, call center analytics, and real-time captioning all require robust STT as the first pipeline stage. Whisper's Apache 2.0 open-source license means you can run it privately on-premise — critical for healthcare, legal, and financial applications where audio data cannot leave your infrastructure. STT is the bridge that connects spoken language to the LLMs that power reasoning and action.

3 Real-World Example

Otter.ai Meeting Intelligence

Otter.ai captures audio from Zoom, Google Meet, and Teams meetings, streams it through an ASR model, performs speaker diarization (labeling who said what), and generates a structured transcript with timestamps. An LLM then produces meeting summaries, action items, and follow-up tasks automatically.

The key engineering challenge is latency: live transcription users expect to see their words within 300-500ms of speaking. This requires streaming ASR — processing audio in overlapping 30-second chunks and emitting partial transcripts as they arrive — rather than waiting for the full recording. Otter.ai processes over 1 billion minutes of audio per year across 100+ countries, making multilingual accuracy and real-time performance existential engineering requirements.

4 Step-by-Step: How Whisper Works

  1. Audio Preprocessing: Raw audio is resampled to 16kHz mono. A 30-second window is extracted. Shorter clips are padded with silence.
  2. Mel Spectrogram: The waveform is converted to a 2D spectrogram — x-axis is time, y-axis is frequency (80 mel bands). Brightness = loudness. This is the model's "image" of the audio.
  3. Audio Encoder: Two CNN layers process the spectrogram, then Transformer encoder layers create 1500 audio feature vectors (512 dims each).
  4. Text Decoder: A Transformer decoder generates tokens auto-regressively. Cross-attention to the encoder features guides each token toward the correct word.
  5. Timestamps: Word-level timestamps are extracted by analyzing decoder attention weights — identifying which audio frames each generated word attended to most.

Speaker Diarization: Whisper alone doesn't label who said what — it just transcribes. The pyannote.audio library handles diarization by clustering audio segments by voice characteristics. Standard pipeline: Whisper transcribes → pyannote diarizes → timestamps aligned to assign each sentence to a speaker label.

5 Python Code Example

python
# pip install faster-whisper openai
# faster-whisper = 4x faster, 4x less VRAM, same accuracy as original
from faster_whisper import WhisperModel

# Model sizes: tiny/base/small/medium/large-v3
# tiny: runs on CPU in real-time   large-v3: GPU needed for speed
model = WhisperModel("base", device="cpu", compute_type="int8")

# Basic transcription with timestamps
segments, info = model.transcribe("meeting.mp3", beam_size=5)
print(f"Language: {info.language} ({info.language_probability:.0%} confident)")

full_text = []
for seg in segments:
    print(f"[{seg.start:.1f}s → {seg.end:.1f}s] {seg.text.strip()}")
    full_text.append(seg.text)
transcript = " ".join(full_text)

# Word-level timestamps (great for subtitle generation)
segments, _ = model.transcribe("audio.mp3", word_timestamps=True)
for seg in segments:
    for word in seg.words:
        print(f"  {word.word!r:20s}  {word.start:.2f}s – {word.end:.2f}s  (conf={word.probability:.2f})")

# OpenAI Whisper API (cloud — no GPU required on your end)
from openai import OpenAI
api_client = OpenAI()

with open("audio.mp3", "rb") as f:
    result = api_client.audio.transcriptions.create(
        model="whisper-1",
        file=f,
        response_format="verbose_json",
        timestamp_granularities=["word"],
        language="en"  # omit for auto-detect
    )
print(result.text)

# Full pipeline: audio → transcript → LLM summary
def transcribe_and_summarize(audio_path: str) -> dict:
    segs, info = model.transcribe(audio_path)
    transcript = " ".join(s.text for s in segs)

    response = api_client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user",
            "content": f"Summarize this transcript. Extract action items.

{transcript}"}]
    )
    return {
        "language": info.language,
        "transcript": transcript,
        "summary": response.choices[0].message.content
    }

result = transcribe_and_summarize("team_meeting.mp3")
print(result["summary"])

7 Practical Use Cases

Meeting intelligence: Record meetings → Whisper transcribes → pyannote diarizes speaker turns → LLM generates summary, action items, and decisions. This is the exact pipeline behind Otter.ai, Fireflies.ai, and Notion AI — multi-million dollar products built on Whisper at their core.

Voice-enabled RAG chatbot: Record user voice query → Whisper transcribes to text → RAG retrieves relevant documents → LLM generates answer → TTS speaks it back. Fully hands-free AI assistant in under 200 lines of Python.

Podcast and video indexing: Transcribe all content → chunk transcripts → embed and index in a vector database → enable semantic search across thousands of hours of audio content with natural language queries.

8 Common Mistakes

❌ Not resampling audio before transcribing: Whisper expects 16kHz mono. Passing 44.1kHz stereo audio causes subtle timing and accuracy errors. Always resample first: ffmpeg -i input.mp4 -ar 16000 -ac 1 output.wav
❌ Using large-v3 for real-time transcription on CPU: large-v3 runs at 0.5× real-time on CPU — 1 hour of audio takes 2 hours to transcribe. For live use, run base on CPU or medium on GPU. Match model size to latency constraints, not just accuracy goals.
⚠️ Whisper hallucination on silence: Whisper can generate plausible text from ambient noise or silence. Always check segment avg_logprob confidence scores. Filter out segments below a threshold (e.g., < -1.0) before passing transcripts to downstream LLMs.
❌ Naive 30-second chunk boundaries: Splitting audio at exactly 30-second boundaries cuts words mid-sentence. Use overlapping chunks and the condition_on_previous_text=True option to maintain coherence across chunk boundaries.
